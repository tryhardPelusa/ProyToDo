Estructura de componentes y plantillas

Mi web está dividida en 3 plantillas. La plantilla de documentación, tareas por orden cronológico y tareas por estado.

He utilizado los siguientes componentes:

  • Tarea: En este componente se muestra todo lo relacionado con una tarea específica. Está añadida dentro de la vista mi-listado. Se compone de título, resumen, fecha y estado.
  • borrado-tarea.service: Esto es un servicio en vez de un componente, sin embargo, lo añado a la lista ya que varios componentes lo han utilizado como auxiliar para funcionar correctamente. Un servicio es una clase que se utiliza para organizar y compartir lógica, datos y funcionalidades entre diferentes componentes de una aplicación. He aprendido a usarlo en la siguiente web: https://desarrolloweb.com/articulos/servicios-angular.html. En principio solo iba a servirme para el borrado de tareas por eso tiene ese nombre, sin embargo, lo he acabado utilizando para cargar el listado de tareas, borrar tareas, añadir tareas y modificar tareas. He preferido no cambiarle el nombre ya que no conozco ninguna herramienta de refactorización en Visual Studio y se me iba a olvidar cambiar alguna referencia.
  • Agregar-tarea: Este componente se encarga de agregar una nueva tarea a la lista. Tiene unos campos en el html en los que se recogen los datos (input) que introduzca el usuario. He tenido aquí un problema con la fecha y hora, ya que de primeras había puesto un campo de tipo date en el html, pero de esta forma no se podía recoger la hora. Asique al final he acabado realizando un selector manual y le he pasado los datos escogidos al constructor Date dentro de typeScript para guardar la fecha y hora al completo. Este componente se apoya en borrado-tarea.service, que es el que se encargará de insertar una nueva tarea al array. Finalmente he guardado la fecha y hora como un número para poder trabajar con él internamente. A la hora de mostrarlo lo he convertido en tipo date.
  • Modificar-tarea: Este componente usa los mismos campos que el componente agregar-tarea. En este caso aparecen ya rellenos con los datos de la tarea seleccionada. Los campos podrán modificarse y al pulsar "modificar tarea" se enviarán los nuevos datos al servicio borrado-tarea para que haga la modificación en la tarea correspondiente.
  • mi-listado: Es la vista en la que se representará el array de tareas en orden cronológico. También tiene un botón que actúa como filtro para mostrar solamente las tareas pendientes.
  • mi-listado-ordenado Igual que el anterior, pero separa las tareas entre las realizadas y las pendientes.
  • mi-menu: En este componente se permite elegir la vista a mostrar, pudiendo escoger la documentación, y las listas de tareas ordenadas por fecha o por estado.
  • itarea: Esta es la interfaz que deben cumplir todas las tareas de la lista. Consta de un título, un resumen, una fecha y un estado.
  • about: Es la vista de la documentación.
